# GameAsset Toolkit 计划评审问题记录

日期：2026-04-18
来源：对 `2026-04-18-game-asset-toolkit-plan.md` 的可执行性评审

## 1. 阻塞级问题

### 1.1 TaskStore 使用内存单例，跨进程不可见
- 位置：Task 5（`TaskStore`）、Task 6（Celery Worker）、Task 8（任务查询）
- 问题：FastAPI 与 Celery Worker 是不同进程，内存 `TaskStore` 不共享。
- 影响：`GET /api/tasks/{id}` 可能无法读取 Worker 更新后的任务状态；状态一致性无法保证。
- 建议：
  - 将任务状态持久化到 Redis（Hash/JSON）或数据库。
  - API 查询与 Worker 更新统一读写同一存储。
  - 内存存储仅用于本地测试桩，不作为真实状态源。

### 1.2 任务提交协议不支持“主文件 + 附加文件”
- 位置：Task 8（`POST /api/tasks`）、Task 12（去水印 `mask_file`）
- 问题：当前接口描述是单文件上传，但去水印处理器需要可选遮罩文件 `mask_file`。
- 影响：P0 功能设计与 API 协议不一致，无法无缝实现手动遮罩模式。
- 建议：
  - `POST /api/tasks` 改为支持多文件字段（如 `input_file` + `extra_files[]`）。
  - `params` 中引用附加文件键（例如 `mask_file_key`）。
  - 后端统一做文件保存映射，处理器通过映射读取附加文件。

## 2. 高优先级问题

### 2.1 测试策略未分层，执行成本和稳定性风险高
- 位置：Task 3/4/12/13/14/15/17/18/19/27/31
- 问题：计划中大量测试默认会调用 rembg、ffmpeg、OpenCV 全链路，耗时且受环境依赖影响。
- 影响：CI 不稳定、开发调试慢、故障定位困难。
- 建议：
  - 单元测试：mock 外部依赖，重点测参数校验和调用流程。
  - 集成测试：保留少量真实样例，使用 `integration` 标记并可选执行。
  - 为 ffmpeg/rembg 相关测试添加环境前置检查与跳过条件。

## 3. 中优先级问题

### 3.1 Task 1 中 `git init` 与当前仓库状态重复
- 位置：Task 1
- 问题：仓库已执行 `git init`，计划项已过时。
- 建议：
  - 将该项标记为已完成或移除，避免执行噪音。

### 3.2 取消任务语义未定义完整
- 位置：Task 8（`DELETE /api/tasks/{id}`）
- 问题：仅写“取消/删除任务”，未定义 Celery revoke、终态状态值、文件清理策略。
- 影响：接口行为不确定，前后端联调易出现歧义。
- 建议：
  - 明确取消流程：`revoke(terminate=...)` 策略。
  - 明确状态：增加 `canceled` 或统一为 `failed` 并附错误码。
  - 明确数据清理：上传文件、输出文件、保留时长与立即清理规则。

## 4. 建议的计划修正顺序
1. 先修正任务状态存储方案（Redis/DB）与任务提交协议（多文件）。
2. 再细化测试分层策略（unit/integration）。
3. 最后补齐取消任务语义与过时项清理。
